렌더링 관련

  • 기존 옵시디언에서 tag{}도 사용하면 렌더링 오류남.
  • html형식은 latex형식($)을 안먹어서 그림 설명은 html로 묶으면 안될거같음.
  • 모든 글에서 문단이 넘어갈 때 줄바꿈(엔터)한번씩 하기(붙어 있어서 가독성 떨어짐, 근데 이건 옵시디언 내에서만)
  • 특정 글 들어가면 약간 스크롤되면서 위 여백을 지워주는?거같은데. 근데 어떤건 좀 더 심하게 내려감. 특히 content/에 있는 파일
  • 글 내부에 내가 작성한거를 어떻게 표기하면 좋을지. 인용문 >처럼 박스쳐지면 좋겠는데 중간에 블록 수식 들어가면 끊김. 언더바 _도 렌더링 깨지고…

사이드바 관련

  • ==블로그 좌측 사이드바가 글이 많아질수록 너무 길어짐. 웹 상단에 헤더 만들어서 대분류 카테고리를 가로로 배치하기.==
    • 헤더로 먼저 대분류를 정하면 사이드바에는 그 대분류에 속한 글만 보이게끔.(이건 죽어도 안됨.)
    • 대신 헤더를 클릭하면 해당 폴더 내 첫 글로 이동시켜서 사이드바가 적절히 이동되게끔.
    • 분류: CV, MML, 수학(내에서 또 분류[선대, 미적분…], 생성 모델, 서재?
  • 작은 창에서 사이드바가 전체 화면을 차지. 사이드바를 열고 닫을 수 있게 수정. 창이 일정 크기보다 작아야만 사이드바를 열고 닫는 토글이 생기거나, 항상 토글이 있게끔 수정.
  • 우측 사이드바 Table of Contents 부분은 해당 header로 이동하는 기능인데 요소의 크기가 작음.
  • 폴더를 열고 닫기말고 항상 열려있고(닫는거 안됨. 그냥 폴더가 아닌 하나의 상위 헤더같이.) 메인과 사이드바가 현재처럼 따로 스크롤할 수 있게끔.
  • 모바일로 보면 사이드바밖에 안보임, 컴퓨터로 봐도 창이 작으면 사이드바가 전체를 가림 -> 토글이 없음.
  • 우측 사이드바 잘려서 나옴.
  • 작은 창에서 헤더랑 메인 화면이 겹침

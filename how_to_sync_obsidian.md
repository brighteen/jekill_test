# 옵시디언(Obsidian) 원본 동기화 및 마이그레이션 가이드

이 가이드는 옵시디언에서 정리가 끝난 원본 노트를 깃허브 저장소로 가져와 웹용으로 최종 변환하는 절차를 설명합니다.

---

## 1. 사전 준비: 파일 복사 (Obsidian → GitHub Repo)

옵시디언 저장소에서 수정이 완료된 파일들을 깃허브 저장소(로컬 PC)의 다음 위치에 덮어씌워 주세요.

### **폴더 매핑 안내**
| 옵시디언 폴더 | 깃허브 저장소 내 소스 폴더 경로 | 비고 |
| :--- | :--- | :--- |
| **Linear Algebra** | `/MML/Part I.../2. Linear Algebra` | MML 교재 원본 |
| **선형대수** | `/선형대수` | 개인 학습 노트 (폴더 없을 시 생성) |
| **미적분** | `/미적분` | |
| **확률과 통계** | `/확률과 통계` | |
| **생성모델** | `/생성모델` | |
| **Memo** | `/Memo` | |

> [!IMPORTANT]
> 반드시 **파일명이 아닌 컨텐츠(내용)가 수정된 원본**을 복사해 주세요. 백링크가 옵시디언에 의해 업데이트된 상태여야 합니다.

---

## 2. 변환 스크립트 실행

파일 복사가 완료되면 터미널(VS Code 내 터미널 등)에서 다음 명령어를 입력합니다.

```powershell
python migrate_new_content.py
```

### **스크립트가 자동으로 수행하는 작업:**
- **수식 보호**: 행렬의 줄바꿈(`\\`)이 깨지지 않도록 자동 이스케이프 처리합니다.
- **백링크 복구**: 파일 이름이 바뀌었더라도 섹션 번호(예: 3.2)를 추적하여 링크를 다시 연결합니다.
- **표 렌더링 수정**: 수식 내의 파이프(`|`) 기호를 해결하여 표가 깨지는 현상을 방지합니다.
- **이미지 최적화**: 모든 이미지를 중앙 정렬하고 너비를 500px로 맞춥니다.

---

## 3. 결과 확인 및 배포

변환이 끝나면 로컬 서버에서 결과를 확인합니다.

1.  **로컬 서버 실행**: `bundle exec jekyll serve`
2.  **브라우저 확인**: `http://localhost:4000`에 접속하여 다음 항목을 점검합니다.
    - [ ] MML 섹션 링크(예: 3.2절)가 정상적으로 연결되는가?
    - [ ] 수식 내의 행렬이 한 줄로 나오지 않고 올바른 형태를 유지하는가?
    - [ ] 이미지가 화면을 뚫고 나가지 않고 중앙에 잘 배치되는가?
3.  **깃허브 푸시**: 모든 것이 완벽하다면 변경 사항을 커밋하고 푸시하여 배포합니다.

---

> [!TIP]
> 만약 변환 후에도 특정 링크가 깨져 있다면, 저(Antigravity)에게 해당 파일명과 섹션 번호를 알려주세요. 즉시 스크립트 로직을 보완해 드립니다.

# 옵시디언(Obsidian) 원본 동기화 및 배포 가이드

이 가이드는 옵시디언에서 작성이 완료된 원본 노트를 로컬 깃허브 저장소로 가져와, 웹용(Jekyll)으로 자동 변환하고 배포하는 **최종 표준 절차**를 설명합니다.

---

## 🚀 3단계 배포 프로세스

### 1단계: 원본 파일 동기화 (Obsidian → GitHub Repo)
옵시디언 보관소(Vault)에서 수정/추가된 파일을 깃허브 저장소의 **원본 폴더**에 덮어씌워 주세요.

| 옵시디언 원본 구분 | 깃허브 저장소 내 소스 폴더 경로 | 비고 |
| :--- | :--- | :--- |
| **MML 교재 내용** | `/MML/` | `Part I`, `Part II` 등 모든 파트 폴더 포함 |
| **개인 학습 노트** | `/Note/` | `1.linear-algebra` 등 카테고리별로 관리 |
| **연구 자료** | `/Research/` | 전문적인 연구 관련 마크다운 |
| **프로필 이미지** | `/me.png` | 메인 인덱스 페이지의 프로필 사진 |

> [!IMPORTANT]
> 반드시 **파일명이 한국어인 원본 상태**로 복사해 주세요. 웹용 제목(영문) 변환은 스크립트가 알아서 처리합니다.

---

### 2단계: 마이그레이션 스크립트 실행
터미널에서 다음 명령어를 입력하여 웹용 변환을 시작합니다.

```powershell
python migrate_new_content.py
```

#### **이 과정에서 자동으로 수행되는 작업:**
- **파일명 영문화(ASCII-fication)**: `선형대수.md` → `linear-algebra.md`와 같이 웹 표준에 맞게 자동 변환하며, 백링크 주소를 모두 자동으로 매핑합니다.
- **이미지 체계화**: 이미지 파일명에 폴더 번호(예: `1_`)를 붙여 정리하고, 모든 이미지를 **중앙 정렬**합니다.
- **수식 레이아웃 최적화**: 
    - 긴 블록 수식에 **가로 스크롤바**를 생성하여 화면 탈출을 방지합니다.
    - 행렬 줄바꿈(`\\`) 및 표 내의 파이프(`|`) 기호 충돌을 해결합니다.
- **Callout 박스**: 옵시디언의 인용구(`>`)를 미려한 디자인의 박스로 변환합니다.

---

### 3단계: 결과 확인 및 실제 배포
1.  **로컬 검증**: `bundle exec jekyll serve` 실행 후 `http://localhost:4000` 접속.
2.  **최종 확인**: 
    - [ ] 이미지가 중앙에 잘 정렬되어 있는가?
    - [ ] 긴 수식에서 가로 스크롤이 작동하는가?
    - [ ] 메인 화면(0_me.png)이 정상적으로 보이는가?
3.  **GitHub 푸시**:
```powershell
git add .
git commit -m "update: 콘텐츠 및 레이아웃 최적화 배포"
git push
```

---

> [!TIP]
> **새로운 한국어 파일명이 잘 안 바뀐다면?**
> `migrate_new_content.py` 상단의 `KOREAN_TO_ENG` 딕셔너리에 원하는 영어 이름을 추가하면 다음에 실행할 때 그 이름으로 변환됩니다.

> [!TIP]
> **파일을 삭제하고 싶을 때**
> 원본 폴더(`Note/`, `MML/`, `Research/`)에서 파일을 지우고 마이그레이션 스크립트를 실행하면, 웹사이트에서도 자동으로 삭제됩니다. 스크립트가 실행될 때마다 기존 결과물(`_notes/` 등)을 초기화하고 **현재 존재하는 원본 파일들만 새로 빌드**하기 때문입니다.

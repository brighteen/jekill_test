# 로컬 서버 실행 및 콘텐츠 업데이트 가이드

이 문서는 새 노트를 추가하거나 기존 내용을 수정할 때 로컬 환경에서 사이트를 확인하는 방법을 설명합니다.

---

## 1. 새 노트 및 이미지 추가

1.  **새 노트**: `_notes/` 폴더 내의 적절한 하위 카테고리에 `.md` 파일을 생성하거나 복사합니다.
2.  **이미지**: `assets/` 폴더에 이미지 파일을 넣습니다. (**주의**: 파일명은 반드시 `영문.png` 등 ASCII 형식을 권장합니다.)

## 2. 마이그레이션 스크립트 실행 (중요)
Jekyll 규격에 맞게 수식, 사진 경로, 중앙 정렬을 자동으로 처리합니다.

```powershell
python migrate_to_jekyll.py
```

## 3. 로컬 서버 실행
수정 사항을 실시간으로 확인하기 위해 서버를 켭니다.

```powershell
# 1. Ruby 경로 추가 (터미널 세션마다 한 번만 실행)
$env:Path = "C:\Ruby34-x64\bin;" + $env:Path

# 2. Jekyll 서버 실행
bundle exec jekyll serve --incremental
```

## 4. 웹 브라우저 확인
서버가 켜지면 아래 주소로 접속합니다.
- **주소**: [http://127.0.0.1:4000/jekill_test/](http://127.0.0.1:4000/jekill_test/)

---

## 💡 팁
- **실시간 반영**: 서버가 켜진 상태에서 `.md` 파일을 수정하고 저장하면, Jekyll이 자동으로 감지하여 페이지를 다시 빌드합니다. (웹 브라우저에서 '새로고침'만 하시면 됩니다.)
- **사진 중앙 정렬**: 스크립트가 자동으로 `<div style="text-align: center;" markdown="1">` 태그를 삽입하므로 별도의 작업이 필요 없습니다.
